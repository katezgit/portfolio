# docs/

Canonical, operator-approved notes. Only finalized artifacts cross into this folder. Drafts and exploration live in `.intermediate/` (gitignored).

## What goes here

- **Positioning + voice** — derived from `strategy/` once stable.
- **Tech stack notes** — prototype defaults, deploy targets, frontend conventions.
- **Cross-case-study patterns** — recurring redesign moves I want to remember (e.g. "how I handle empty states", "table density principles") once I've shipped enough to see a pattern. Don't write these speculatively.
- **Outreach assets** — finalized copy variants used in cold outreach, once tested.

## What does NOT go here

- WIP drafts, exploration, debug screenshots → `.intermediate/`
- Per-startup case studies → `case-studies/{slug}/`
- Pitch / positioning sources still in flux → `strategy/`
- Manuals for the operator → `manuals/`
